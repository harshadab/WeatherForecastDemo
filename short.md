# AI Tool Usage (Disclosure)

The purpose of this assignment is to demonstrate object-oriented analysis and modelling, Kotlin coding skills, and Android project structuring. As allowed by the assignment guidelines, AI tools were used as supportive assistants to improve productivity while keeping the focus on understanding product logic and implementation.

## Tools Used
- **ChatGPT**
- **Claude**
- **Android Studio AI features**

## How They Helped

### 1) Design / Architecture Support (ChatGPT, Claude)
- Used during early design to validate and iterate on the overall structure (Clean Architecture layering, separation of concerns, module boundaries).
- Helped brainstorm UI state modelling approaches and identify edge cases (loading/error/empty states).
- Used as a second opinion on tradeoffs and potential improvements (scalability, maintainability).

### 2) Tooling & Refactoring Assistance (Android Studio AI)
- Used for small IDE-assisted refactors such as converting hardcoded UI strings into `strings.xml`.
- Used to assist in resolving Gradle/build configuration issues more quickly (dependency or configuration troubleshooting).

## What Was Kept Under Manual Ownership
- All final architectural decisions (layer boundaries, module structure, dependency direction).
- All core implementation logic (domain models, use cases, repository design, ViewModel/state handling).
- All integration decisions (API mapping, error handling strategy).
- All testing decisions and verification (tests were written/adjusted and executed manually).

## Validation
All code changes—whether AI-suggested or not—were reviewed, adapted where necessary, and verified via local builds and tests to ensure correctness and alignment with the assignment requirements.

## Notes
AI tools were used as productivity accelerators (similar to IDE suggestions and documentation lookup). The focus remained on demonstrating understanding of the product logic and implementation rather than handcrafting every line from scratch.
